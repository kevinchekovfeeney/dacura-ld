view()

view(  $part = "") : \an

GET /[$part]

Fetches the configuration settings for the current collection context

Parameters

$part

optional 'part' of the configuration - can be list for a list of collections (only in the 'all' context)

Returns

\an —

associative array with the fields: settings, services, collection (and meta & servicemeta for all) or a simple array with elements having (status, id, name) when the argument is 'list'

This function does two things to get around the problem that all requests are to the root resource normally (because the arguments are contained in the preceding parts of the url)
